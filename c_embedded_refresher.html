The Core Rule

Two's complement is how virtually all modern CPUs store negative integers. The most significant bit (MSB) is the sign bit (1 = negative). To negate a number: flip all bits, then add 1.

WHY THIS MATTERS IN EMBEDDED Hardware can use the same addition circuit for both signed and unsigned math. A single ADD instruction handles both — the hardware doesn't care; you tell the compiler the intent via the type.

Why int Has No Fixed Size

The C standard says int should be the "natural word size" of the processor — whatever size the CPU handles most efficiently in a single instruction. This made sense in 1972 when C was written for one machine. It became a portability nightmare once C ran on everything from 8-bit AVRs to 64-bit ARM servers.

THE ONLY GUARANTEE THE C STANDARD MAKES
sizeof(char) ≤ sizeof(short) ≤ sizeof(int) ≤ sizeof(long) ≤ sizeof(long long)
It only guarantees relationships between types, not absolute sizes. A conforming compiler could make int 16 bits or 32 bits and be equally correct.

inttypes.h — Portable printf Format Macros

You cannot use %d to print a uint32_t portably. On AVR, %d expects 16-bit int. Use <inttypes.h> macros which expand to the correct specifier per platform:

#include <stdint.h>
#include <inttypes.h>
#include <stdio.h>

uint32_t sensor_val  = 1048576UL;
int32_t  temperature = -45;
uint64_t timestamp   = 1700000000000ULL;

/* ── WRONG: undefined behavior if int is 16-bit on AVR ── */

/* ── CORRECT: PRI macros expand to the right specifier ── */
printf("%" PRIu32 "\n", sensor_val);   /* "u" on ARM, "lu" on AVR    */
printf("%" PRId32 "\n", temperature);  /* signed decimal for int32_t */
printf("%" PRIu64 "\n", timestamp);    /* "llu" on most platforms    */
printf("%" PRIx32 "\n", sensor_val);   /* hex lowercase              */
printf("%" PRIX32 "\n", sensor_val);   /* hex UPPERCASE              */

What Is a Bitmask?

A bitmask is an integer value used as a pattern to isolate, set, clear, or toggle specific bits inside another integer — without disturbing the other bits. In embedded systems, hardware peripheral registers are controlled entirely through bitmasks. Every GPIO pin, every UART flag, every clock enable bit is a single bit inside a 32-bit register.

THE FUNDAMENTAL IDEA A 32-bit register like GPIOA_ODR controls 16 GPIO output pins at once — each bit = one pin. You never write the whole register (that clobbers all other pins). You touch exactly one bit and leave the rest exactly as they were.

Operation 1 — SET a Bit  |=
GPIOA_ODR |= GPIO_PIN_5;    /* set bit 5 — turn LED ON */

Bitwise OR — a result bit is 1 if either input bit is 1. ORing with the mask forces that bit to 1, leaves all others untouched.

GPIOA_ODR = …1111 0000 0010 1010 (bit5=0)
GPIO_PIN_5 = …0000 0000 1010 0000 (mask)
OR result = …1111 0000 1010 1010 (bit5 forced to 1, rest unchanged ✓)
RULE ORing with 1 always sets that bit to 1. ORing with 0 always leaves that bit alone.
Operation 2 — CLEAR a Bit  ~ &=
GPIOA_ODR &= ~GPIO_PIN_5;    /* clear bit 5 — turn LED OFF */

Two operators work together. ~ (bitwise NOT) flips every bit of the mask first, making all bits 1 except bit 5. Then &= (bitwise AND) keeps all existing bits intact — except bit 5 which is ANDed with 0 and forced to 0.

GPIO_PIN_5 = …0000 0000 1010 0000
~GPIO_PIN_5 = …1111 1111 0101 1111 (all 1s except bit 5)
GPIOA_ODR = …1111 0000 1010 1010 (bit5=1)
AND result = …1111 0000 0010 1010 (bit5 forced to 0, rest unchanged ✓)
RULE ANDing with 1 always leaves that bit alone. ANDing with 0 always clears that bit to 0. The ~ flips the mask so only the target bit becomes 0.
Operation 3 — TOGGLE a Bit  ^=
GPIOA_ODR ^= GPIO_PIN_5;    /* toggle bit 5 — blink LED */

XOR (exclusive OR) — result is 1 if inputs are different. XORing with 1 flips the bit; XORing with 0 leaves it alone. Every call toggles: ON→OFF→ON→OFF.

Call 1 (bit5=0): …0000 0000 0010 0000 ^ 1010 0000 = 1010 0000 (LED ON)
Call 2 (bit5=1): …0000 0000 1010 0000 ^ 1010 0000 = 0010 0000 (LED OFF)
RULE XORing with 1 always flips (toggles) that bit. XORing with 0 always leaves that bit alone. This is the foundation of all embedded LED blink code.
Operation 4 — READ / TEST a Bit  &
/* Test if bit 5 is set */
if (GPIOA_IDR & GPIO_PIN_5) { /* nonzero = bit is SET */ }

/* Get exact 0 or 1 */
uint8_t pin5_state = (GPIOA_IDR >> 5) & 1U;

/* Store as bool */
bool led_is_on = (GPIOA_ODR & GPIO_PIN_5) != 0;
WHY THE RESULT IS NOT JUST 0 OR 1 (GPIOA_IDR & GPIO_PIN_5) gives 0x00000020 (= 32), not 1 — but in C, any nonzero value is truthy in an if(). If you need the literal 0 or 1, shift right: (GPIOA_IDR >> 5) & 1U.

The Five Memory Regions — Process Address Space

Every C program's memory is divided into five distinct regions. On a bare-metal MCU (e.g. STM32) the layout is fixed by the linker script. On Linux/desktop, the OS maps these into virtual address space. From lowest to highest address:

STACK
grows ↓ downward · local vars · frames
↕ free space (unmapped)
HEAP
grows ↑ upward · malloc/free · dynamic
BSS
zero-initialized globals & statics · no binary size cost
DATA
initialized globals & statics · stored in binary
TEXT (Code)
compiled instructions · read-only · lowest address
#include <stdio.h>
#include <stdlib.h>

int global_init   = 42;        /* DATA segment  — initialized global    */
int global_uninit;              /* BSS  segment  — zero at startup       */
static int s_count = 0;        /* BSS  segment  — static, zero-inited   */
static int s_val   = 7;        /* DATA segment  — static, non-zero      */

void foo(void) {               /* TEXT segment  — machine instructions  */
    int local = 5;              /* STACK         — lives on stack frame  */
    int *p = malloc(4);         /* HEAP          — p itself on stack, *p on heap */
    free(p);
}
TEXT Segment — Code / Instructions Read-Only · Flash on MCU

Contains the compiled machine instructions of your program. On embedded systems this lives in Flash memory (ROM). It is read-only at runtime — writing to it causes a hard fault. String literals (e.g. "hello") are also stored here.

/* These all live in TEXT / Flash */
void add(int a, int b) { return a + b; }   /* compiled to machine opcodes */

const char *msg = "hello";       /* "hello" bytes stored in TEXT */
msg[0] = 'H';                    /* CRASH — writing to read-only memory! */

/* To modify: copy to RAM first */
char buf[] = "hello";            /* array on STACK — bytes copied from TEXT to RAM */
buf[0] = 'H';                    /* fine — stack is writable */
EMBEDDED: FLASH vs RAM On an STM32, Flash starts at 0x08000000 and SRAM at 0x20000000. Your linker script maps TEXT to Flash, DATA/BSS/Stack/Heap to SRAM. A const global with no __attribute__((section)) goes to Flash automatically.
DATA Segment — Initialized Globals & Statics Stored in Binary

Holds all global and static variables that have a non-zero initial value. Their initial values are stored in the binary (Flash on MCU). At startup, the C runtime copies them from Flash into RAM. This copy is done by the startup.s or CRT0 code before main() runs.

/* All of these go to DATA segment */
int          g_speed    = 100;      /* global, non-zero init value  */
static float s_ratio    = 3.14f;   /* file-scope static, non-zero  */
char         g_mode     = 'A';      /* char with init value         */
uint32_t     g_bitmask  = 0xDEAD;   /* non-zero hex literal         */

void foo(void) {
    static int call_count = 1;      /* local static, non-zero → DATA */
    call_count++;                    /* persists between calls       */
}

⚠ EMBEDDED FLASH COST Every initialized global takes space in your Flash binary. A uint8_t buf[1024] = {0xFF} adds 1 KB to your .bin file. If you initialize to zero, use BSS instead — it costs no Flash.
BSS Segment — Zero-Initialized Globals & Statics No Binary Size Cost

Block Started by Symbol — holds globals and statics with value 0 or no explicit initializer. BSS takes up no space in your binary — the linker just records the size. The C runtime zeroes it at startup with a memset-style loop.

/* All go to BSS — cost 0 bytes in the .bin file */
int      g_counter;              /* no initializer → BSS, starts as 0    */
int      g_flags   = 0;         /* explicit zero  → BSS                 */
char     g_buf[256];            /* large zero buffer → BSS (no Flash cost!) */
static uint32_t s_state;        /* file-scope static, no init → BSS     */

void foo(void) {
    static int seen = 0;         /* local static, zero init → BSS        */
    if (!seen) { seen = 1; init(); } /* run-once pattern */
}

int x = 1; → DATA again
HEAP — Dynamic Memory malloc · free · grows upward

The heap is a pool of RAM managed at runtime by the allocator (malloc/free). It grows upward from a base address. On bare-metal MCUs the heap is a fixed region defined in the linker script — there is no OS to expand it. Fragmentation and leaks are permanent until reset.

#include <stdlib.h>

/* ── Basic allocation ── */
int *arr = (int*)malloc(10 * sizeof(int));  /* 40 bytes from heap */
if (arr == NULL) { /* always check — heap can be full */ }
arr[0] = 42;
free(arr);                                   /* return to heap     */
arr = NULL;                                   /* prevent use-after-free */

/* ── calloc — allocate + zero-init ── */
int *buf = (int*)calloc(10, sizeof(int));   /* 10 ints, all zero  */

/* ── realloc — resize existing allocation ── */
arr = (int*)realloc(arr, 20 * sizeof(int));  /* grow to 20 ints    */
if (arr == NULL) { /* original block still valid if realloc fails */ }

/* ── Heap layout after three malloc calls ── */
// [heap base] [block A: 40B][block B: 16B][block C: 100B] [free ...]
// After free(B): [block A][FREE: 16B][block C] — fragmented!
⚠ EMBEDDED: AVOID HEAP IN RTOS/ISR malloc is not ISR-safe (uses a lock internally). On FreeRTOS use pvPortMalloc/vPortFree. On truly resource-constrained MCUs, avoid dynamic allocation entirely and use static pools instead.
Stack Frames — Push, Pop, Layout grows downward

Every function call pushes a stack frame (also called an activation record) onto the stack. The frame holds: local variables, saved registers, the return address, and passed arguments (those that don't fit in registers). When the function returns the frame is popped — the stack pointer moves back up.

/* ── This code: ── */
int add(int a, int b) {
    int result = a + b;   /* local var — lives in this frame */
    return result;
}

int multiply(int x, int y) {
    int temp = add(x, y);  /* calls add — pushes new frame   */
    return temp * 2;
}

int main(void) {
    int r = multiply(3, 4);  /* pushes multiply frame          */
}

Stack state at deepest point (inside add):

HIGH ADDRESS (stack base)
main frame — r (unset), saved regs, return addr to OS
multiply frame — x=3, y=4, temp (unset), saved regs, return addr → main
add frame — a=3, b=4, result=7, saved regs, return addr → multiply
← SP (stack pointer) points here
LOW ADDRESS (stack grows downward)

Each frame in detail (ARM Cortex-M / x86-64 style):

/* What the compiler emits for add() — conceptually: */

/* PROLOGUE — push frame */
// PUSH {lr}          save return address (link register)
// SUB  sp, sp, #8    reserve 8 bytes for locals (result + alignment)
// STR  r0, [sp, #0]  store arg a
// STR  r1, [sp, #4]  store arg b

/* BODY */
// LDR  r0, [sp, #0]
// LDR  r1, [sp, #4]
// ADD  r0, r0, r1    result = a + b  (answer in r0)

/* EPILOGUE — pop frame */
// ADD  sp, sp, #8    restore stack pointer (locals gone)
// POP  {pc}          restore return address → jumps back to caller
⚠ STACK OVERFLOW ON MCU STM32F4 default stack is 512 bytes. A recursive function or large local array can overflow it silently — corrupting heap or BSS. Always define stack size explicitly in the linker script and use a stack canary or FreeRTOS stack high-water mark to detect overflow.
Where Does the Return Value Go? Registers · Not Stack

Return values are passed back through CPU registers, not the stack. On ARM (AAPCS calling convention): small values go in r0, 64-bit values in r0:r1, and structs larger than 8 bytes via a hidden pointer argument. On x86-64: rax for integers, xmm0 for floats.

/* ── Small return (int, pointer, bool) → r0 on ARM ── */
int get_count(void) {
    return 42;          /* compiler: MOV r0, #42 ; BX lr */
}
int c = get_count();   /* c = r0 = 42                   */

/* ── 64-bit return → r0:r1 pair on ARM32 ── */
uint64_t get_timestamp(void) {
    return 0x0000000100000002ULL;
    /* r0 = low  32 bits (0x00000002) */
    /* r1 = high 32 bits (0x00000001) */
}

/* ── Struct return — small (≤8 bytes) → r0:r1 ── */
typedef struct { int x; int y; } Point;
Point make_point(void) {
    return (Point){3, 4};  /* r0 = x=3, r1 = y=4 */
}

/* ── Large struct — caller allocates space, passes hidden ptr ── */
typedef struct { int data[10]; } BigStruct;
BigStruct make_big(void) {        /* compiler rewrites as: */
    BigStruct s = {{0}};          /* void make_big(BigStruct *ret) */
    return s;                      /* writes directly into caller's memory */
}
ARM AAPCS CALLING CONVENTION SUMMARY Arguments: first 4 in r0–r3, rest on stack. Return value: r0 (32-bit), r0:r1 (64-bit), or hidden pointer for large structs. Callee must preserve r4–r11. Caller owns r0–r3 (scratch). This is why passing large structs by value is expensive — the compiler inserts hidden copies.
volatile — Prevent Compiler Optimization Critical in Embedded

volatile tells the compiler: "this variable can change at any time outside normal program flow" — do not cache it in a register, do not optimize away reads or writes, re-read it from memory every single time. Used for hardware registers, ISR-shared flags, and memory-mapped I/O.

Problem Without volatile — Optimizer Breaks Your Code
/* ── WITHOUT volatile — the optimizer sees this: ── */
int flag = 0;            /* ISR sets this to 1       */

while (flag == 0) {}     /* PROBLEM: compiler loads flag into register ONCE */
                          /* sees it never changes in this loop body */
                          /* optimizes to: while(true) {} — infinite loop! */

/* What GCC -O2 actually emits without volatile: */
// LDR r0, [flag]   ← load once
// CMP r0, #0
// BEQ .Linfinite   ← branch to infinite loop — ISR update never seen!

/* ── WITH volatile — correct behavior: ── */
volatile int flag = 0;

while (flag == 0) {}     /* re-reads flag from RAM every iteration */
// LDR r0, [flag]   ← load on EVERY loop iteration
// CMP r0, #0
// BEQ .Lloop       ← loops and re-reads — ISR update seen correctly!
Use Case 1 — ISR Flag (most common embedded pattern)
/* ── ISR sets flag, main loop polls it ── */
volatile bool g_uart_rx_ready = false;

/* ISR — runs asynchronously, interrupts main at any time */
void USART1_IRQHandler(void) {
    g_uart_rx_ready = true;   /* write must reach RAM, not just register */
}

/* Main loop — must see the ISR's write */
while (!g_uart_rx_ready) {}  /* without volatile: optimizer kills this loop */
process_uart_data();
g_uart_rx_ready = false;
Use Case 2 — Memory-Mapped Hardware Registers
/* ── STM32 GPIO — read hardware register ── */
/* WITHOUT volatile: compiler may cache the register value! */
uint32_t *reg = (uint32_t*)(0x40020010);   /* GPIOA_IDR address */
while (*reg & (1U << 13)) {}                /* wait for button — may never update! */

/* WITH volatile: every read goes to the actual hardware register */
volatile uint32_t *reg = (volatile uint32_t*)(0x40020010);
while (*reg & (1U << 13)) {}    /* reads hardware every iteration — correct! */

/* CMSIS style (used in all STM32 HAL code): */
while (!(GPIOA->IDR & GPIO_IDR_ID13)) {}   /* CMSIS defines all regs as volatile */
Use Case 3 — Delay Loops (prevent optimization)
/* ── Without volatile: compiler deletes the loop entirely ── */
for (int i = 0; i < 1000; i++) {}       /* "does nothing" → deleted at -O1+ */

/* ── With volatile: forces every iteration to execute ── */
for (volatile int i = 0; i < 1000; i++) {}  /* loop body preserved */

/* Better practice: use a hardware timer or SysTick, not a delay loop */
Use Case 4 — volatile with structs and pointers
/* ── volatile pointer to volatile data ── */
volatile uint32_t * volatile p;
//  ^data is volatile      ^pointer itself is volatile

/* ── volatile struct member ── */
typedef struct {
    volatile uint32_t CR1;   /* each register read goes to hardware */
    volatile uint32_t CR2;
    volatile uint32_t SR;
} USART_TypeDef;

volatile USART_TypeDef *USART1 = (USART_TypeDef*)(0x40013800);
USART1->CR1 |= (1U << 13);   /* enable USART — must hit hardware */

/* ── volatile const — read-only hardware register (e.g. status) ── */
volatile const uint32_t *STATUS_REG = (volatile const uint32_t*)(0x40020000);
uint32_t s = *STATUS_REG;   /* reads hardware; compiler can't write it */
⚠ volatile ≠ atomic ≠ thread-safe volatile only prevents the compiler from caching a value. It does NOT prevent CPU out-of-order execution, does NOT provide atomicity on multi-core systems, and does NOT act as a memory barrier. For shared state between OS threads, use _Atomic or a mutex. For ISR↔main on a single-core MCU, volatile is usually sufficient for simple flags — but for compound operations (read-modify-write), you still need to disable interrupts around the critical section.

strlen size_t

Returns the number of characters excluding the null terminator. Undefined behavior if passed NULL or a non-null-terminated buffer — it will walk memory forever until it finds a \0.

size_t strlen(const char *str);

size_t len  = strlen("hello");   /* 5 — excludes \0            */
size_t zero = strlen("");        /* 0                          */
size_t bad  = strlen(NULL);      /* UB! crash — never do this  */
⚠ SIZE ISSUE No size parameter — will walk memory indefinitely if no \0 is found. Always ensure the string is properly null-terminated before calling.
strcpy & strncpy char*

strcpy copies src into dest including the null terminator. No bounds checking — if dest is too small, it silently overflows and corrupts memory. strncpy adds a size limit but has its own trap: it does not guarantee null termination if src is longer than n.

char *strcpy (char *dest, const char *src);
char *strncpy(char *dest, const char *src, size_t n);

/* strcpy — overflow danger */
char dest[6];
strcpy(dest, "hello");        /* ok — "hello\0" fits in 6   */
strcpy(dest, "toolongstring"); /* overflow! silent corruption */

/* strncpy — must manually null-terminate */
char buf[5];
strncpy(buf, "hello", 5);     /* "hello" — NO \0 added!     */
buf[4] = '\0';                /* always add this manually   */

strncpy(buf, "hi", 5);        /* "hi\0\0\0" — pads rest     */
⚠ SIZE ISSUE — strncpy If src length ≥ n, the result has no null terminator. Always add dest[n-1] = '\0' after strncpy. Prefer snprintf for safe string building.
strcat & strncat char*

Appends src to the end of dest. strcat has no bounds checking. strncat limits how many chars from src are appended and always adds a null terminator — but you must calculate n correctly.

char *strcat (char *dest, const char *src);
char *strncat(char *dest, const char *src, size_t n);

char dest[12] = "hello";
strcat(dest, " world");         /* ok — dest = "hello world"  */
strcat(dest, "!!!!!!");         /* overflow! dest too small   */

/* strncat — n = remaining space - 1 */
char safe[12] = "hello";
strncat(safe, " world!!!!!", sizeof(safe) - strlen(safe) - 1);
/* appends up to 6 chars, always null-terminates */
SIZE REQUIREMENT strcat needs dest to have at least strlen(dest) + strlen(src) + 1 bytes allocated. Always compute the required size before calling.
strcmp & strncmp int

Compares strings lexicographically. Returns 0 if equal, <0 if s1 comes before s2, >0 if s1 comes after s2. The exact non-zero values are not guaranteed to be 1 or -1.

int strcmp (const char *s1, const char *s2);
int strncmp(const char *s1, const char *s2, size_t n);

strcmp("abc", "abc");    /*  0  — equal                    */
strcmp("abc", "abd");    /* <0  — 'c' < 'd'               */
strcmp("abd", "abc");    /* >0  — 'd' > 'c'               */
strcmp("abc", "abcd");   /* <0  — shorter string is "less" */

/* CORRECT comparison */
if (strcmp(s1, s2) == 0)  { /* equal ✓     */ }
if (!strcmp(s1, s2))       { /* equal ✓     */ }
if (strcmp(s1, s2) == 1)  { /* WRONG ✗ — only guaranteed <0, 0, >0 */ }

/* strncmp — compare only first n chars (prefix check) */
strncmp("hello world", "hello there", 5);  /* 0 — first 5 chars equal */
strchr · strrchr · strstr char* or NULL

Search functions — all return a pointer into the original string on success, or NULL if not found. Always check for NULL before dereferencing the result.

char *strchr (const char *str, int c);         /* first occurrence of char  */
char *strrchr(const char *str, int c);         /* last  occurrence of char  */
char *strstr (const char *hay, const char *needle); /* first occurrence of substr */

char *str = "hello world";

char *p = strchr(str, 'l');        /* points to "llo world"      */
char *q = strrchr(str, 'l');       /* points to "ld"             */
char *r = strstr(str, "world");    /* points to "world"          */
char *x = strchr(str, 'z');        /* NULL — not found           */

/* always check NULL before use! */
if (p != NULL) {
    int index = p - str;             /* get index: pointer arithmetic */
    printf("%d\n", index);          /* 2                          */
}

if (r != NULL)
    printf("%s\n", r);             /* "world"                    */
⚠ NULL DEREFERENCE Dereferencing the result of these functions without checking for NULL is one of the most common C bugs. Always guard with if (p != NULL).
sprintf & snprintf int

Format integers, floats, or strings into a char buffer. sprintf has no bounds checking. snprintf is the safe version — always use it. The return value tells you how many chars would have been written; if it's ≥ size, the output was truncated.

int sprintf (char *str, const char *fmt, ...);
int snprintf(char *str, size_t size, const char *fmt, ...);

/* sprintf — dangerous, no bounds check */
char small[4];
sprintf(small, "%d", 123456);   /* overflow! silent corruption */

/* snprintf — safe, always null-terminates */
char buf[8];
int n = snprintf(buf, sizeof(buf), "hello world");
/* buf = "hello w\0" — truncated! */
/* n = 11 — what it WOULD have written */

if (n >= (int)sizeof(buf)) {
    /* handle truncation — always check! */
}
✓ GOLDEN RULE Always use snprintf over sprintf. Always check if the return value ≥ size. This is the only way to detect truncation.
strtok char* or NULL

Splits a string into tokens by delimiter. Two critical gotchas: it modifies the original string (replaces delimiters with \0) and uses internal static state — making it not thread-safe.

char *strtok(char *str, const char *delim);

char str[] = "hello,world,foo";   /* MUST be mutable — not a literal! */
char *token = strtok(str, ",");    /* first call: pass the string      */
while (token != NULL) {
    printf("%s\n", token);          /* hello, world, foo                */
    token = strtok(NULL, ",");      /* subsequent calls: pass NULL      */
}
/* str is now "hello\0world\0foo" — original destroyed! */

/* Thread-safe alternative (POSIX) */
char *saveptr;
token = strtok_r(str, ",", &saveptr);
while (token != NULL)
    token = strtok_r(NULL, ",", &saveptr);
⚠ THREAD SAFETY Never use strtok in multithreaded code (RTOS tasks, pthreads). Use strtok_r on POSIX or implement your own tokenizer with an explicit state pointer.
memcpy · memmove · memset void*

Raw memory operations — work on bytes, not strings. No null terminator logic. memcpy is fast but undefined behavior on overlapping regions. memmove is safe for overlap. memset fills memory with a byte value.

void *memcpy (void *dest, const void *src, size_t n);
void *memmove(void *dest, const void *src, size_t n);
void *memset (void *ptr,  int c,           size_t n);

/* memcpy — no overlap allowed */
char src[] = "hello";
char dst[6];
memcpy(dst, src, 6);              /* copies 6 bytes incl. \0    */
memcpy(src+1, src, 4);           /* UB! regions overlap        */

/* memmove — handles overlap safely */
char buf[] = "hello";
memmove(buf+1, buf, 4);          /* safe: "hhell\0"            */

/* memset — common patterns */
char zeros[32];
memset(zeros, 0,   sizeof(zeros)); /* zero buffer — safe        */
memset(zeros, 'a', sizeof(zeros)); /* fill with 'a'             */

/* memset gotcha: only safe for 0 with integer arrays */
int arr[10];
memset(arr, 0, sizeof(arr));      /* ok — 0x00000000 = 0       */
memset(arr, 1, sizeof(arr));      /* WRONG! sets bytes to 1, not ints to 1 */

Declaration, Initialization & Memory Layout

A 2D array in C is not an array of pointers. It is a single contiguous block of memory laid out in row-major order — all elements of row 0 first, then row 1, etc. This is the single most important fact about multidimensional arrays in C.

/* ── Declaration ── */
int a[3][4];                     /* 3 rows, 4 columns — uninitialized (stack) */
int b[3][4] = {0};               /* all zeros */
int c[3][4] = {{1,2,3,4},       /* row 0 */
               {5,6,7,8},       /* row 1 */
               {9,10,11,12}};   /* row 2 */

/* ── Memory layout: ALL 12 ints are contiguous ── */
/* Address: [base+0] [base+4] [base+8] [base+12] | [base+16] ... */
/*           c[0][0]  c[0][1]  c[0][2]  c[0][3]  | c[1][0] ...  */

/* ── Accessing elements ── */
int val = c[1][2];   /* = 7   — row 1, col 2 */
/* Compiler computes: base + (row * COLS + col) * sizeof(int)    */
/* = base + (1*4 + 2)*4 = base + 24                              */

/* ── 3D array ── */
int d[2][3][4];   /* 2 blocks, each 3 rows × 4 cols = 24 ints contiguous */
/* d[b][r][c] → base + (b*3*4 + r*4 + c) * sizeof(int)          */

Array Decay — The Most Common Interview Trap

When a 2D array name is used in an expression, it decays to a pointer to its first row — a pointer to an array of COLS ints, not a pointer to a pointer (int**). This is the #1 source of confusion and wrong function signatures.

int c[3][4];

/* What 'c' decays to: */
int (*row_ptr)[4] = c;    /* pointer to array of 4 ints — CORRECT type */
int **ptr2ptr    = c;    /* WRONG — compiler warning/error, NOT equivalent */

/* sizeof does NOT decay — it sees the full array: */
printf("%zu\n", sizeof(c));          /* 48 — full array size */
printf("%zu\n", sizeof(row_ptr));    /* 4 or 8 — pointer size only! */

/* Pointer arithmetic on row_ptr advances by one FULL ROW (16 bytes): */
row_ptr++;   /* now points to c[1] — advanced by sizeof(int[4]) = 16 bytes */

/* Three equivalent ways to access c[1][2] = 7: */
c[1][2];               /* normal indexing */
(*(c + 1))[2];         /* c+1 = row 1, then index col 2 */
*(*( c + 1) + 2);      /* fully dereferenced */
*(((int*)c) + 1*4 + 2); /* flat pointer arithmetic — same address */
⚠ INTERVIEW TRAP: int** vs int(*)[N] int** is a pointer to a pointer — it points to a list of separate row pointers (like a dynamically allocated jagged array). int(*)[4] is a pointer to a contiguous 4-element row — what a real 2D array decays to. Passing a 2D array to a function expecting int** is a type error and likely a crash.
Passing 2D Arrays to Functions — All Valid Forms

This is the most commonly tested topic in C interviews. You must specify the column dimension so the compiler can compute row strides. The row dimension is optional (and ignored).

/* ── Form 1: Explicit 2D array parameter (col dimension required) ── */
void print_matrix(int arr[3][4]) {          /* rows=3 is ignored */
    for (int i=0; i<3; i++)
        for (int j=0; j<4; j++)
            printf("%d ", arr[i][j]);
}

/* ── Form 2: Pointer to row array — most explicit, same thing ── */
void print_matrix(int (*arr)[4], int rows) { /* pass rows separately */
    for (int i=0; i<rows; i++)
        for (int j=0; j<4; j++)
            printf("%d ", arr[i][j]);
}

/* ── Form 3: C99 VLA — runtime dimensions ── */
void print_matrix(int rows, int cols, int arr[rows][cols]) {
    for (int i=0; i<rows; i++)
        for (int j=0; j<cols; j++)
            printf("%d ", arr[i][j]);
}
/* rows and cols must come BEFORE arr in the parameter list (C99) */

/* ── Form 4: Flat pointer (manual index math) ── */
void print_matrix(int *arr, int rows, int cols) {
    for (int i=0; i<rows; i++)
        for (int j=0; j<cols; j++)
            printf("%d ", arr[i*cols + j]);  /* manual row-major math */
}
print_matrix((int*)c, 3, 4);   /* cast to int* on call — works because contiguous */

bad((int**)c);                          /* type mismatch AND undefined behavior */

int  a[3][4];

/* Q: What does sizeof give for each? */
sizeof(a)          /* = 48  — 3 × 4 × sizeof(int) = 3×4×4 */
sizeof(a[0])       /* = 16  — one row: 4 × sizeof(int) */
sizeof(a[0][0])    /* = 4   — one element */
sizeof(p)          /* = 8   — pointer size (64-bit system) */
sizeof(*p)         /* = 16  — what p points to: int[4] */

/* Computing dimensions from sizeof: */
int rows = sizeof(a) / sizeof(a[0]);        /* 48/16 = 3 */
int cols = sizeof(a[0]) / sizeof(a[0][0]); /* 16/4  = 4 */

/* ONLY works on the actual array, NOT on a pointer/parameter! */
void foo(int arr[3][4]) {
    sizeof(arr);   /* = 8 — sizeof(int(*)[4]), pointer! Not 48. */
}
⚠ sizeof decays inside functions Once an array is passed to a function, sizeof gives the pointer size, not the array size. This is why you always pass the dimensions separately.
Classic Interview Algorithms on 2D Arrays
Matrix Transpose — In-place (square) & Out-of-place
/* In-place transpose — only works for square (N×N) */
#define N 3
void transpose_inplace(int a[N][N]) {
    for (int i=0; i<N; i++)
        for (int j=i+1; j<N; j++) {  /* j starts at i+1 — only upper triangle */
            int t = a[i][j];
            a[i][j] = a[j][i];
            a[j][i] = t;
        }
}
/* Time: O(n²)  Space: O(1) */

/* Out-of-place transpose — works for any M×N → N×M */
void transpose(int src[3][4], int dst[4][3]) {
    for (int i=0; i<3; i++)
        for (int j=0; j<4; j++)
            dst[j][i] = src[i][j];
}
/* Time: O(m×n)  Space: O(m×n) */
Rotate Matrix 90° Clockwise (in-place, N×N)
/* Step 1: transpose. Step 2: reverse each row. */
void rotate_90(int a[N][N]) {
    /* transpose */
    for (int i=0; i<N; i++)
        for (int j=i+1; j<N; j++) {
            int t=a[i][j]; a[i][j]=a[j][i]; a[j][i]=t;
        }
    /* reverse each row */
    for (int i=0; i<N; i++)
        for (int j=0, k=N-1; j<k; j++,k--) {
            int t=a[i][j]; a[i][j]=a[i][k]; a[i][k]=t;
        }
}
/* 180°: rotate_90 twice.  270°: transpose then reverse each column. */
/* Time: O(n²)  Space: O(1) */
Spiral Order Traversal

/* Start at TOP-RIGHT corner: if val > target → move left; if val < target → move down */
int search_sorted_matrix(int a[4][4], int rows, int cols, int target) {
    int r=0, c=cols-1;
    while (r<rows && c>=0) {
        if      (a[r][c] == target) return  1;   /* found */
        else if (a[r][c]  > target) c--;          /* too big — go left */
        else                        r++;          /* too small — go down */
    }
    return -1;   /* not found */
}

Set Row & Column to Zero (in-place, O(1) space)
/* If a[i][j]==0, set entire row i and col j to 0 */
/* Naive: use O(m+n) extra arrays. Optimal: use first row/col as flags */
void zero_matrix(int a[3][4], int R, int C) {
    int row0=0, col0=0;
    for (int j=0; j<C; j++) if (a[0][j]==0) row0=1;   /* does row 0 need zeroing? */
    for (int i=0; i<R; i++) if (a[i][0]==0) col0=1;   /* does col 0 need zeroing? */
    for (int i=1; i<R; i++)                            /* use row0/col0 as marker */
        for (int j=1; j<C; j++)
            if (a[i][j]==0) { a[i][0]=0; a[0][j]=0; }
    for (int i=1; i<R; i++) if (a[i][0]==0) for(int j=0;j<C;j++) a[i][j]=0;
    for (int j=1; j<C; j++) if (a[0][j]==0) for(int i=0;i<R;i++) a[i][j]=0;
    if (row0) for(int j=0;j<C;j++) a[0][j]=0;
    if (col0) for(int i=0;i<R;i++) a[i][0]=0;
}

Cache Performance — Row-Major vs Column-Major Access

Because C arrays are row-major, iterating column by column (inner loop over rows) causes a cache miss on every single element access. This can make your code 10–100× slower on large matrices. This is a classic interview performance question.

int a[1000][1000];

/* ── FAST: row-major — accesses consecutive memory addresses ── */
for (int i=0; i<1000; i++)         /* outer = rows    */
    for (int j=0; j<1000; j++)     /* inner = columns */
        a[i][j] = i + j;
/* a[i][0], a[i][1], a[i][2]... — sequential, prefetcher loves this */

/* ── SLOW: column-major — jumps 4000 bytes per access ── */
for (int j=0; j<1000; j++)         /* outer = columns */
    for (int i=0; i<1000; i++)     /* inner = rows    */
        a[i][j] = i + j;
/* a[0][j], a[1][j], a[2][j]... — strides by 4000 bytes, cache miss every time */

/* ── Cache-oblivious matrix multiply (tiling / blocking) ── */
#define BLOCK 32
for (int ii=0; ii<N; ii+=BLOCK)
  for (int jj=0; jj<N; jj+=BLOCK)
    for (int kk=0; kk<N; kk+=BLOCK)
      for (int i=ii; i<ii+BLOCK && i<N; i++)
        for (int j=jj; j<jj+BLOCK && j<N; j++)
          for (int k=kk; k<kk+BLOCK && k<N; k++)
            C[i][j] += A[i][k] * B[k][j];
/* Processes 32×32 tiles that fit in L1 cache — dramatically faster */

Member Access: . vs ->

. accesses a member of a struct value (the struct itself is in scope). -> is shorthand for dereferencing a pointer to a struct and then accessing the member. They are completely equivalent — ptr->member is exactly (*ptr).member.

typedef struct {
    int   x;
    int   y;
    float speed;
} Particle;

/* ── Dot operator: struct value on the stack ── */
Particle p = {10, 20, 3.5f};
p.x = 15;            /* direct member access */
p.speed += 1.0f;
printf("%d\n", p.y);

/* ── Arrow operator: pointer to struct ── */
Particle *pp = &p;
pp->x = 30;          /* identical to: (*pp).x = 30 */
pp->speed = 5.0f;    /* identical to: (*pp).speed = 5.0f */

/* ── Proof they're the same — all four access y identically: ── */
p.y;                  /* dot on value */
(&p)->y;              /* arrow on pointer to value */
(*pp).y;              /* deref pointer then dot */
pp->y;                /* arrow — most readable */

/* ── Heap-allocated struct — always use arrow ── */
Particle *hp = (Particle*)malloc(sizeof(Particle));
hp->x = 0;  hp->y = 0;  hp->speed = 1.0f;
free(hp);

Use -> when you have a pointer to a struct — from malloc, passed as argument, or taken with &.

In practice: function parameters are almost always pointers (to avoid copying), so -> dominates in real code.

How the Compiler Inserts Padding

CPUs read memory most efficiently when data is naturally aligned — a 4-byte int at an address divisible by 4. The compiler pads structs to enforce this.

/* ── BAD layout: 8 bytes of data → 12 bytes allocated ── */
typedef struct {
    uint8_t  a;       /* offset 0, size 1 */
    /* [3 bytes padding] */
    uint32_t b;       /* offset 4, size 4 */
    uint8_t  c;       /* offset 8, size 1 */
    /* [3 bytes padding] */
} Bad_t;              /* sizeof(Bad_t) = 12 */

/* ── GOOD layout: same data, sorted large→small → 8 bytes ── */
typedef struct {
    uint32_t b;       /* offset 0, size 4 */
    uint8_t  a;       /* offset 4, size 1 */
    uint8_t  c;       /* offset 5, size 1 */
    /* [2 bytes padding] */
} Good_t;             /* sizeof(Good_t) = 8 */

/* ── PACKED: zero padding — for hardware register maps & protocols ── */
typedef struct __attribute__((packed)) {
    uint8_t  cmd;     /* offset 0 */
    uint16_t length;  /* offset 1 — unaligned! may be slow on ARM without unaligned access */
    uint32_t payload; /* offset 3 — unaligned */
} __attribute__((packed)) UartPacket_t;  /* sizeof = 7, matches wire format exactly */

/* ── Check layout at compile time ── */
_Static_assert(sizeof(UartPacket_t) == 7, "Packet size mismatch");
_Static_assert(offsetof(UartPacket_t, length) == 1, "Offset wrong");

Pointer Arithmetic — The Complete Picture High Interview Weight

Pointer arithmetic is not byte arithmetic. When you add n to a pointer, the address advances by n × sizeof(*pointer). The compiler automatically scales the offset by the pointed-to type's size. This is the most commonly tested pointer topic in embedded and systems interviews.

/* ── Core rule: p + n advances by n × sizeof(*p) bytes ── */
uint8_t  *p8  = (uint8_t  *)0x1000;
uint16_t *p16 = (uint16_t *)0x1000;
uint32_t *p32 = (uint32_t *)0x1000;

p8  + 1  /* = 0x1001  — advanced by 1 byte  (sizeof uint8_t  = 1) */
p16 + 1  /* = 0x1002  — advanced by 2 bytes (sizeof uint16_t = 2) */
p32 + 1  /* = 0x1004  — advanced by 4 bytes (sizeof uint32_t = 4) */

p8  + 3  /* = 0x1003  — 3×1  = 3 bytes */
p16 + 3  /* = 0x1006  — 3×2  = 6 bytes */
p32 + 3  /* = 0x100C  — 3×4 = 12 bytes */

/* ── Interview question: what is the output? ── */
int arr[] = {10, 20, 30, 40, 50};
int *p = arr;
printf("%d\n", *(p + 2));    /* 30 — p+2 skips 2×4=8 bytes → arr[2] */
printf("%d\n", *p + 2);     /* 12 — (*p)+2 = 10+2. Precedence: * binds tighter than + */
printf("%d\n", *(p++));     /* 10 — reads *p first, THEN increments p */
printf("%d\n", *(++p));     /* 30 — p was at arr[1] from above, pre-inc → arr[2] */
printf("%d\n", *p++);      /* 30 — *p++ same as *(p++): read then advance */
PRECEDENCE TRAP: *p++ vs (*p)++ vs *(p++) *p++ — read *p, then advance p (most common in loops)
(*p)++ — increment the value at p, pointer unchanged
*(p++) — identical to *p++: ++ post-fix binds to p, not *p
*(++p) — advance p first, then read new location

Rule: postfix ++ and [] bind tighter than unary *.

sizeof — Array vs Pointer Decay Most Common Interview Bug

An array name used in most expressions decays to a pointer to its first element, losing all size information. sizeof is one of the very few exceptions — it sees the real array type. But once the array is passed to a function, the parameter receives a pointer, and sizeof gives pointer width, not array size.

/* ── On the actual array: sizeof works correctly ── */
uint8_t buf[256];
sizeof(buf)         /* = 256 — compiler sees the full array type */
sizeof(buf[0])      /* = 1   — size of one element */
sizeof(buf) / sizeof(buf[0])  /* = 256 — idiomatic element count */

/* ── Once a pointer is taken, sizeof gives pointer width ── */
uint8_t *p = buf;
sizeof(p)           /* = 8 on 64-bit, 4 on 32-bit — pointer size, NOT 256! */
sizeof(*p)          /* = 1 — size of what p points to (uint8_t) */

/* ── Function parameter — array decays to pointer ── */
void process(uint8_t *buf) {
    sizeof(buf);      /* = 8 (pointer width) — NOT the buffer size! */
}
void process2(uint8_t buf[256]) {   /* [256] is silently ignored — same as uint8_t* */
    sizeof(buf);      /* still 8 — the [256] decorates only the source */
}

What Is a Double Pointer?

A double pointer (int **pp) is a pointer that holds the address of another pointer. Every level of * adds one level of indirection. The classic real-world example is char **argv — a pointer to an array of pointers, each pointing to a string.

/* ── Single pointer ── */
int  x   = 42;
int *p   = &x;     /* p   holds address of x */
*p = 99;           /* writes 99 into x via p */

/* ── Double pointer ── */
int **pp = &p;     /* pp  holds address of p */
**pp = 7;          /* writes 7  into x via pp→p→x */
*pp  = NULL;       /* sets p  to NULL via pp */

/* Memory picture: */
/* [pp @ 0x300] → [p @ 0x200] → [x @ 0x100] = 42 */
/*  value:0x200    value:0x100   value:42          */

/* ── Why double pointers are needed ── */

What Is a Ring Buffer and Why Use It?

A ring buffer (also called circular buffer or FIFO) is a fixed-size buffer that treats its memory as if the ends are connected — when the write pointer reaches the end of the array, it wraps back to the start. It is the single most important data structure in embedded systems because it enables safe, zero-copy data transfer between an ISR producer and a main-loop consumer without needing a mutex.

KEY PROPERTIES Fixed memory — no malloc, no fragmentation.
O(1) push and pop.
Lock-free when single producer + single consumer (SPSC).
FIFO ordering — first in, first out.
Data is never moved — only indices advance.
CLASSIC USE CASES UART RX ISR → main loop processing.
ADC DMA samples → DSP algorithm.
Logging: background writer, foreground reader.
CAN / SPI message queues between tasks.
Audio codec double-buffering.
Why the MASK? Power-of-2 Size & Bitwise Wrap

Index wrap-around is required when the head or tail reaches the end of the buffer. There are two ways to do it. Understanding the difference is a critical interview topic.

/* ── Method 1: Modulo (%) — works for ANY size ── */
head = (head + 1) % SIZE;   /* e.g. (63+1)%64 = 0 — wraps correctly */
/* Problem: % is a division instruction — slow on MCUs without hardware divide */
/* ARM Cortex-M0 has no hardware divide — % compiles to a software div routine */
/* ~20 cycles on Cortex-M0 vs 1 cycle for & */

/* ── Method 2: Bitmask (&) — ONLY works when SIZE is a power of 2 ── */
#define SIZE  64U           /* power of 2: 2,4,8,16,32,64,128,256... */
#define MASK  (SIZE - 1U)  /* = 0x3F = 0b00111111 */
head = (head + 1) & MASK;  /* = (head+1) & 63 — clears bits above bit 5 */

/* WHY THIS WORKS — binary demonstration: */
/* SIZE = 64 = 0b01000000                  */
/* MASK = 63 = 0b00111111  (SIZE - 1)      */
/*                                         */
/* head = 62: (62+1) & 63 = 63 & 63 = 63  */
/* head = 63: (63+1) & 63 = 64 & 63       */
/*             = 0b01000000                */
/*             & 0b00111111                */
/*             = 0b00000000 = 0  ← wraps! */
/*                                         */
/* For ANY power-of-2 N: x & (N-1) == x % N */
/* This equivalence ONLY holds for powers of 2. For SIZE=60, MASK=59 is WRONG. */

/* ── How to verify SIZE is a power of 2 at compile time ── */
#define IS_POWER_OF_2(n)  (((n) & ((n)-1)) == 0)
_Static_assert(IS_POWER_OF_2(SIZE), "Ring buffer size must be power of 2");
⚠ INTERVIEW TRAP: Non-power-of-2 size with MASK Using & (SIZE-1) when SIZE is not a power of 2 silently produces wrong wrap-around. Example: SIZE=60, MASK=59=0b00111011. Index 60 wraps to 60&59=56, not 0. Always use _Static_assert or a compile-time check.
Full Production Implementation
#include <stdint.h>
#include <stdbool.h>
#include <string.h>

#define RB_SIZE   256U                  /* MUST be power of 2 */
#define RB_MASK  (RB_SIZE - 1U)         /* 0xFF */

typedef struct {
    uint8_t           buf[RB_SIZE];
    volatile uint32_t head;   /* producer writes here (use uint32_t for atomic write on ARM) */
    volatile uint32_t tail;   /* consumer reads from here */
} RingBuf_t;

/* ── Init ── */
void rb_init(RingBuf_t *rb) {
    rb->head = 0;
    rb->tail = 0;
    memset(rb->buf, 0, sizeof(rb->buf));
}

/* ── Empty / Full / Count ── */
static inline bool     rb_empty(const RingBuf_t *rb) { return rb->head == rb->tail; }
static inline bool     rb_full (const RingBuf_t *rb) { return ((rb->head + 1) & RB_MASK) == rb->tail; }
static inline uint32_t rb_count(const RingBuf_t *rb) { return (rb->head - rb->tail) & RB_MASK; }
static inline uint32_t rb_free (const RingBuf_t *rb) { return RB_MASK - rb_count(rb); }
/* Note: maximum usable capacity = SIZE-1 (one slot always kept empty to distinguish full vs empty) */

/* ── Push (called from ISR or producer) ── */
bool rb_push(RingBuf_t *rb, uint8_t byte) {
    uint32_t next_head = (rb->head + 1) & RB_MASK;
    if (next_head == rb->tail) return false;   /* full — drop byte */
    rb->buf[rb->head] = byte;
    rb->head = next_head;                        /* head updated AFTER write */
    return true;
}

/* ── Pop (called from main loop or consumer) ── */
bool rb_pop(RingBuf_t *rb, uint8_t *out) {
    if (rb->tail == rb->head) return false;    /* empty */
    *out = rb->buf[rb->tail];
    rb->tail = (rb->tail + 1) & RB_MASK;       /* tail updated AFTER read */
    return true;
}

/* ── Peek (read without consuming) ── */
bool rb_peek(const RingBuf_t *rb, uint8_t *out) {
    if (rb_empty(rb)) return false;
    *out = rb->buf[rb->tail];
    return true;
}

/* ── Push block (copy n bytes in) ── */
uint32_t rb_push_block(RingBuf_t *rb, const uint8_t *src, uint32_t len) {
    uint32_t written = 0;
    while (written < len && rb_push(rb, src[written])) written++;
    return written;   /* may be less than len if buffer fills */
}

/* ── Pop block (drain up to n bytes) ── */
uint32_t rb_pop_block(RingBuf_t *rb, uint8_t *dst, uint32_t len) {
    uint32_t read = 0;
    while (read < len && rb_pop(rb, &dst[read])) read++;
    return read;
}
⚠ WHY HEAD IS UPDATED AFTER WRITE (critical ordering) In the SPSC scenario: the ISR writes data then updates head. If head were updated before writing the data, the consumer could read stale/garbage data from the new head slot before the ISR writes it. Always: write data first, then advance the index. A memory barrier (__DMB()) may be needed on out-of-order CPUs (not required on Cortex-M).
Practical Example — UART RX ISR + Main Loop Parser
/* ── Global ring buffer for UART1 RX ── */
static RingBuf_t uart_rx_buf;

/* ── ISR: called automatically on each received byte ── */
void USART1_IRQHandler(void) {
    if (USART1->SR & USART_SR_RXNE) {             /* byte ready? */
        uint8_t byte = (uint8_t)(USART1->DR & 0xFF);
        if (!rb_push(&uart_rx_buf, byte)) {
            error_flags |= ERR_UART_OVERFLOW;     /* buffer full — log overrun */
        }
    }
}

/* ── Main loop: drain ring buffer and parse NMEA sentences ── */
static char    line[128];
static uint8_t line_idx = 0;

    uint8_t byte;
    while (rb_pop(&uart_rx_buf, &byte)) {

            line[line_idx] = '\0';
            parse_sentence(line);               /* process complete line */
            line_idx = 0;
        } else {
            line[line_idx++] = (char)byte;
        }
    }
}

    uart_init(115200);           /* enables RXNE interrupt */

        uart_process();           /* non-blocking drain */

Full vs Empty — The Lost-Slot Problem

A fundamental ambiguity: when head == tail, does that mean empty or full? You cannot distinguish them with just two equal indices. There are three standard solutions, each with trade-offs.

/* ── Solution 1: Keep one slot empty (used above) ── */
/* Empty: head == tail                                    */
/* Full:  (head+1)&MASK == tail                           */
/* Capacity: SIZE-1 (wastes 1 slot but simplest code)     */

/* ── Solution 2: Separate count field ── */
typedef struct {
    uint8_t           buf[RB_SIZE];
    volatile uint32_t head;
    volatile uint32_t tail;
    volatile uint32_t count;   /* explicit occupancy counter */
} RingBufFull_t;
/* Empty: count==0   Full: count==SIZE   Capacity: SIZE   */
/* ⚠ count must be updated atomically — not lock-free     */

/* ── Solution 3: Mirror bit (unbounded indices) ── */
/* head and tail are never masked — allowed to grow freely  */
/* Empty: head == tail                                       */
/* Full:  head - tail == SIZE                                */
/* Count: head - tail (works even after wrap due to unsigned)*/
/* Capacity: SIZE (all slots usable)                         */
uint32_t count = rb->head - rb->tail;   /* correct due to unsigned wrap */
uint8_t *slot  = &rb->buf[rb->head & RB_MASK];  /* mask ONLY on buffer index */
✓ BEST PRACTICE: Use unbounded indices (Solution 3) Store raw incrementing head/tail (uint32_t), apply mask only when indexing into the buffer array. Gives full SIZE capacity, clean count calculation, and remains lock-free SPSC. This is what Linux kernel ring buffers and DPDK use.

    int         data;
    struct Node *next;

Node* new_node(int val) {
    Node *n = (Node*)malloc(sizeof(Node));
    n->data = val;
    n->next = NULL;
    return n;
}

/* ── Insert at head — O(1) ── */
Node* insert_head(Node *head, int val) {
    Node *n = new_node(val);
    n->next = head;
    return n;               /* return new head */
}

/* ── Insert at tail — O(n) ── */
Node* insert_tail(Node *head, int val) {
    Node *n = new_node(val);
    if (!head) return n;
    Node *cur = head;
    while (cur->next) cur = cur->next;
    cur->next = n;
    return head;
}

/* ── Insert at position k (0-indexed) ── */
Node* insert_at(Node *head, int val, int k) {
    if (k == 0) return insert_head(head, val);
    Node *cur = head;
    for (int i=0; i<k-1 && cur; i++) cur = cur->next;
    if (!cur) return head;  /* k out of range */
    Node *n = new_node(val);
    n->next = cur->next;
    cur->next = n;
    return head;
}

/* ── Delete by value — O(n) ── */
Node* delete_val(Node *head, int val) {
    if (!head) return NULL;
    if (head->data == val) {
        Node *tmp = head->next;
        free(head);
        return tmp;
    }
    Node *cur = head;
    while (cur->next && cur->next->data != val) cur = cur->next;
    if (cur->next) {
        Node *tmp = cur->next;
        cur->next = tmp->next;
        free(tmp);
    }
    return head;
}

/* ── Print list ── */
void print_list(Node *head) {
    while (head) { printf("%d→", head->data); head = head->next; }
    printf("NULL\n");
}

/* ── Free entire list ── */
void free_list(Node *head) {
    while (head) { Node *tmp = head; head = head->next; free(tmp); }
}

Reverse a Linked List — Iterative & Recursive
/* Iterative — O(n) time, O(1) space — preferred */
Node* reverse(Node *head) {
    Node *prev = NULL, *cur = head, *next = NULL;
    while (cur) {
        next      = cur->next;  /* save next */
        cur->next = prev;       /* reverse link */
        prev      = cur;        /* advance prev */
        cur       = next;       /* advance cur */
    }
    return prev;   /* new head */
}

/* Recursive — O(n) time, O(n) stack space */
Node* reverse_rec(Node *head) {
    if (!head || !head->next) return head;
    Node *new_head = reverse_rec(head->next);
    head->next->next = head;
    head->next = NULL;
    return new_head;
}

/* Reverse between positions left and right (1-indexed) */
Node* reverse_between(Node *head, int left, int right) {
    if (!head || left == right) return head;
    Node dummy = {0, head};
    Node *pre = &dummy;
    for (int i=1; i<left; i++) pre = pre->next;
    Node *cur = pre->next;
    for (int i=0; i<right-left; i++) {
        Node *nxt = cur->next;
        cur->next = nxt->next;
        nxt->next = pre->next;
        pre->next = nxt;
    }
    return dummy.next;
}
Detect & Remove Cycle (Floyd's Algorithm)
/* Floyd's tortoise and hare — O(n) time, O(1) space */
bool has_cycle(Node *head) {
    Node *slow = head, *fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) return true;   /* met inside cycle */
    }
    return false;
}

/* Find the start node of the cycle */
Node* cycle_start(Node *head) {
    Node *slow = head, *fast = head, *entry = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
        if (slow == fast) {
            while (entry != slow) { entry = entry->next; slow = slow->next; }
            return entry;   /* cycle start */
        }
    }
    return NULL;
}

/* Remove cycle (once start is found) */
void remove_cycle(Node *head) {
    Node *start = cycle_start(head);
    if (!start) return;
    Node *cur = start;
    while (cur->next != start) cur = cur->next;
    cur->next = NULL;
}

/* Returns middle node — for odd length: exact middle, even: second middle */

Merge Two Sorted Lists
Node* merge_sorted(Node *a, Node *b) {
    Node dummy = {0, NULL};
    Node *tail = &dummy;
    while (a && b) {
        if (a->data <= b->data) { tail->next = a; a = a->next; }
        else                     { tail->next = b; b = b->next; }
        tail = tail->next;
    }
    tail->next = a ? a : b;   /* attach remaining */
    return dummy.next;
}

Find Nth Node from End
Node* nth_from_end(Node *head, int n) {
    Node *ahead = head, *behind = head;
    for (int i=0; i<n; i++) {
        if (!ahead) return NULL;   /* n > length */
        ahead = ahead->next;
    }
    while (ahead) { ahead = ahead->next; behind = behind->next; }
    return behind;
}

Check Palindrome
bool is_palindrome(Node *head) {
    Node *mid  = find_middle(head);
    Node *rev  = reverse(mid);       /* reverse second half */
    Node *p1   = head, *p2 = rev;
    bool  ok   = true;
    while (p2) {
        if (p1->data != p2->data) { ok = false; break; }
        p1 = p1->next; p2 = p2->next;
    }
    reverse(rev);   /* restore list */
    return ok;
}
Intersection Point of Two Lists
Node* intersection(Node *a, Node *b) {
    Node *pa = a, *pb = b;
    while (pa != pb) {
        pa = pa ? pa->next : b;   /* when pa exhausted, restart at b */
        pb = pb ? pb->next : a;   /* when pb exhausted, restart at a */
    }
    return pa;   /* both traverse same total distance: lenA+lenB */
}
/* If no intersection, both become NULL simultaneously — returns NULL */

typedef struct DNode {

void dll_insert_after(DNode *node, int val) {

    n->data       = val;
    n->next       = node->next;
    n->prev       = node;
    if (node->next) node->next->prev = n;
    node->next    = n;
}

/* Delete a node — O(1) given the pointer (vs O(n) for singly linked) */
void dll_delete(DNode *node) {
    if (node->prev) node->prev->next = node->next;
    if (node->next) node->next->prev = node->prev;
    free(node);
}
/* Key advantage: O(1) delete when you have the node pointer */
/* Used in: LRU cache, browser history, text editor undo     */

    if (q_full(q)) return false;
    q->data[q->rear] = val;
    q->rear = (q->rear + 1) % Q_CAP;
    q->size++;
    return true;
}

    if (q_empty(q)) return false;
    *out = q->data[q->front];
    q->front = (q->front + 1) % Q_CAP;
    q->size--;
    return true;
}

Sliding Window Maximum (Monotonic Deque)
/* Find max in each window of size k — O(n) using deque of indices */
void sliding_max(int *a, int n, int k, int *out) {
    int deq[1024], front=0, back=0;    /* deque holds indices */
    for (int i=0; i<n; i++) {
        while (front<back && deq[front] < i-k+1) front++;   /* evict old */
        while (front<back && a[deq[back-1]] <= a[i]) back--;  /* evict smaller */
        deq[back++] = i;
        if (i >= k-1) out[i-k+1] = a[deq[front]];           /* front = max */
    }
}

BST Node & Core Operations
typedef struct TNode {
    int          val;
    struct TNode *left, *right;
} TNode;

static TNode* tnode(int v) {
    TNode *n = (TNode*)malloc(sizeof(TNode));
    n->val=v; n->left=n->right=NULL;
    return n;
}

/* ── BST Insert — O(h) h=height ── */
TNode* bst_insert(TNode *root, int v) {
    if (!root) return tnode(v);
    if (v < root->val) root->left  = bst_insert(root->left,  v);
    else if (v > root->val) root->right = bst_insert(root->right, v);
    return root;   /* duplicate ignored */
}

/* ── BST Search — O(h) ── */
TNode* bst_search(TNode *root, int v) {
    if (!root || root->val==v) return root;
    return v < root->val ? bst_search(root->left,v) : bst_search(root->right,v);
}

/* ── BST Delete — 3 cases ── */
TNode* bst_min(TNode *n) { while(n->left) n=n->left; return n; }

TNode* bst_delete(TNode *root, int v) {
    if (!root) return NULL;
    if      (v < root->val) root->left  = bst_delete(root->left,  v);
    else if (v > root->val) root->right = bst_delete(root->right, v);
    else {
        if  (!root->left)  { TNode *t=root->right; free(root); return t; }
        if  (!root->right) { TNode *t=root->left;  free(root); return t; }
        TNode *succ = bst_min(root->right);       /* in-order successor */
        root->val   = succ->val;
        root->right = bst_delete(root->right, succ->val);
    }
    return root;
}

/* Height — O(n) */
int height(TNode *r) {
    if (!r) return 0;
    int l = height(r->left), rh = height(r->right);
    return 1 + (l > rh ? l : rh);
}

/* Check balanced (|left_height - right_height| <= 1 at every node) */
int check_balanced(TNode *r) {   /* returns -1 if unbalanced, else height */
    if (!r) return 0;
    int l = check_balanced(r->left);
    int rh = check_balanced(r->right);
    if (l==-1 || rh==-1 || (l-rh)>1 || (rh-l)>1) return -1;
    return 1 + (l > rh ? l : rh);
}
bool is_balanced(TNode *r) { return check_balanced(r) != -1; }

/* Diameter — longest path between any two nodes (may not pass through root) */
int g_diam = 0;
int diam_helper(TNode *r) {
    if (!r) return 0;
    int l = diam_helper(r->left), rh = diam_helper(r->right);
    if (l+rh > g_diam) g_diam = l+rh;
    return 1 + (l>rh?l:rh);
}
Lowest Common Ancestor (LCA)
/* LCA for BST — O(h) */
TNode* lca_bst(TNode *root, int p, int q) {
    if (!root) return NULL;
    if (p < root->val && q < root->val) return lca_bst(root->left,  p, q);
    if (p > root->val && q > root->val) return lca_bst(root->right, p, q);
    return root;   /* root is between p and q — it's the LCA */
}

/* LCA for any binary tree — O(n) */
TNode* lca_bt(TNode *root, int p, int q) {
    if (!root || root->val==p || root->val==q) return root;
    TNode *l = lca_bt(root->left,  p, q);
    TNode *r = lca_bt(root->right, p, q);
    if (l && r) return root;   /* p and q are in different subtrees */
    return l ? l : r;
}

/* ── Bubble Sort — with early-exit optimization ── */
void bubble_sort(int *a, int n) {
    for (int i=0; i<n-1; i++) {
        bool swapped = false;
        for (int j=0; j<n-i-1; j++) {
            if (a[j] > a[j+1]) {
                int t=a[j]; a[j]=a[j+1]; a[j+1]=t;
                swapped = true;
            }
        }
        if (!swapped) break;   /* already sorted — O(n) best case */
    }
}

/* ── Selection Sort — fewest swaps (useful when write cost is high) ── */
void selection_sort(int *a, int n) {
    for (int i=0; i<n-1; i++) {
        int min_idx = i;
        for (int j=i+1; j<n; j++)
            if (a[j] < a[min_idx]) min_idx = j;
        if (min_idx != i) { int t=a[i]; a[i]=a[min_idx]; a[min_idx]=t; }
    }
}
/* Good for flash memory writes (EEPROM) where write cycles are limited */

/* ── Insertion Sort — best for small or nearly-sorted arrays ── */
void insertion_sort(int *a, int n) {
    for (int i=1; i<n; i++) {
        int key = a[i], j = i-1;
        while (j>=0 && a[j]>key) { a[j+1]=a[j]; j--; }
        a[j+1] = key;
    }
}

Merge Sort — Guaranteed O(n log n)
void merge(int *a, int l, int m, int r) {
    int n1=m-l+1, n2=r-m;
    int L[n1], R[n2];
    for(int i=0;i<n1;i++) L[i]=a[l+i];
    for(int j=0;j<n2;j++) R[j]=a[m+1+j];
    int i=0,j=0,k=l;
    while(i<n1&&j<n2) a[k++]=(L[i]<=R[j])?L[i++]:R[j++];
    while(i<n1) a[k++]=L[i++];
    while(j<n2) a[k++]=R[j++];
}
void merge_sort(int *a, int l, int r) {
    if(l<r){
        int m=l+(r-l)/2;
        merge_sort(a,l,m); merge_sort(a,m+1,r); merge(a,l,m,r);
    }
}

Quick Sort — Fastest in Practice
int partition(int *a, int lo, int hi) {
    int pivot = a[hi], i = lo-1;
    for(int j=lo;j<hi;j++)
        if(a[j]<=pivot){i++; int t=a[i];a[i]=a[j];a[j]=t;}
    int t=a[i+1];a[i+1]=a[hi];a[hi]=t;
    return i+1;
}
void quick_sort(int *a, int lo, int hi) {
    if(lo<hi){
        int p=partition(a,lo,hi);
        quick_sort(a,lo,p-1); quick_sort(a,p+1,hi);
    }
}

/* ── Median-of-3 pivot — avoids O(n²) on sorted input ── */
int median3(int *a, int lo, int hi) {
    int mid = lo+(hi-lo)/2;
    if(a[lo]>a[mid]){int t=a[lo];a[lo]=a[mid];a[mid]=t;}
    if(a[lo]>a[hi]) {int t=a[lo];a[lo]=a[hi]; a[hi]=t;}
    if(a[mid]>a[hi]){int t=a[mid];a[mid]=a[hi];a[hi]=t;}
    return mid;   /* median now at mid, place at hi as pivot */
}

/* ── 3-way partition (Dutch National Flag) — handles duplicates well ── */
void quick3(int *a, int lo, int hi) {
    if(lo>=hi) return;
    int lt=lo, gt=hi, i=lo, pivot=a[lo], t;
    while(i<=gt){
        if     (a[i]<pivot){t=a[lt];a[lt++]=a[i];a[i++]=t;}
        else if(a[i]>pivot){t=a[gt];a[gt--]=a[i];a[i]=t;}
        else i++;
    }
    quick3(a,lo,lt-1); quick3(a,gt+1,hi);  /* [lt..gt] are all == pivot */
}
Heap Sort — In-Place O(n log n) Guaranteed
void heapify(int *a, int n, int i) {
    int largest=i, l=2*i+1, r=2*i+2;
    if(l<n && a[l]>a[largest]) largest=l;
    if(r<n && a[r]>a[largest]) largest=r;
    if(largest!=i){
        int t=a[i];a[i]=a[largest];a[largest]=t;
        heapify(a,n,largest);
    }
}
void heap_sort(int *a, int n) {
    for(int i=n/2-1;i>=0;i--) heapify(a,n,i);   /* build max-heap: O(n) */
    for(int i=n-1;i>0;i--){
        int t=a[0];a[0]=a[i];a[i]=t;   /* move max to end */
        heapify(a,i,0);                   /* re-heapify reduced heap */
    }
}
/* O(1) extra space. Guaranteed O(n log n) — no pathological input.  */

Counting Sort & Radix Sort — O(n) for Integers
/* ── Counting Sort — O(n+k), stable, integers in range [0,k) ── */
void counting_sort(int *a, int n, int k) {
    int *count = (int*)calloc(k, sizeof(int));
    int *out   = (int*)malloc(n * sizeof(int));
    for(int i=0;i<n;i++) count[a[i]]++;           /* frequency */
    for(int i=1;i<k;i++) count[i]+=count[i-1];   /* prefix sum */
    for(int i=n-1;i>=0;i--) out[--count[a[i]]]=a[i];  /* place (stable) */
    for(int i=0;i<n;i++) a[i]=out[i];
    free(count); free(out);
}
/* Best when k (range) is small compared to n (number of elements)    */

/* ── Radix Sort — sorts by digit, uses counting sort as subroutine ── */
void count_by_digit(int *a, int n, int exp) {
    int out[n], count[10]={0};
    for(int i=0;i<n;i++) count[(a[i]/exp)%10]++;
    for(int i=1;i<10;i++) count[i]+=count[i-1];
    for(int i=n-1;i>=0;i--) out[--count[(a[i]/exp)%10]]=a[i];
    for(int i=0;i<n;i++) a[i]=out[i];
}
void radix_sort(int *a, int n) {
    int mx=a[0];
    for(int i=1;i<n;i++) if(a[i]>mx) mx=a[i];
    for(int exp=1; mx/exp>0; exp*=10)
        count_by_digit(a, n, exp);
}
/* O(n * d) where d = number of digits in max element            */
/* For 32-bit ints: d=10, so effectively O(10n) = O(n)           */

Linked list: mergesort (no random access).

Recursion — When It's Safe in Embedded
⚠ EMBEDDED CAUTION Embedded stacks are small (often 1–4KB). Deep recursion causes stack overflow with no exception — just corrupt memory. Prefer iteration. Use recursion only when stack depth is bounded and known. 
/* ── Safe: bounded recursion — binary search (log₂(N) depth) ── */
int32_t bsearch_reg(const uint16_t *arr, int32_t lo, int32_t hi, uint16_t target) {
    if (lo > hi) return -1;
    int32_t mid = lo + (hi - lo) / 2;
    if (arr[mid] == target) return mid;
    if (arr[mid] < target)  return bsearch_reg(arr, mid+1, hi, target);
    return bsearch_reg(arr, lo, mid-1, target);
}   /* max depth = log₂(N): for N=1024, max 10 frames */

/* ── Tail recursion (compiler converts to loop if -O2) ── */
uint32_t factorial(uint32_t n, uint32_t acc) {
    if (n == 0) return acc;
    return factorial(n-1, n * acc);   /* tail call */
}

Most common on embedded Linux (Raspberry Pi, i.MX). Full POSIX threading with mutex, semaphores, conditions.

ISO C11 standard — portable but not always supported on bare-metal toolchains. GCC supports it.

POSIX pthreads — Create, Start, End, Sync
#include <pthread.h>
#include <stdatomic.h>    /* C11 atomics */
#include <stdio.h>

/* ── Shared data (must be protected) ── */
static pthread_mutex_t s_lock = PTHREAD_MUTEX_INITIALIZER;
static uint32_t        s_counter = 0;       /* guarded by s_lock */
static _Atomic uint32_t s_atomic_cnt = 0;  /* C11 atomic — no lock needed */

/* ── Thread function signature: void* fn(void*) ── */
void* worker_thread(void *arg) {
    uint32_t id = (uint32_t)(uintptr_t)arg;

    for (int i = 0; i < 1000; i++) {
        /* mutex-protected shared counter */
        pthread_mutex_lock(&s_lock);
        s_counter++;
        pthread_mutex_unlock(&s_lock);

        /* OR: C11 atomic (no mutex needed, lock-free) */
        atomic_fetch_add(&s_atomic_cnt, 1);
    }
    pthread_exit(NULL);   /* thread end */
    return NULL;          /* alternative to pthread_exit */
}

int main(void) {
    pthread_t tid[4];

    /* ── Thread start ── */
    for (int i = 0; i < 4; i++) {
        pthread_create(&tid[i], NULL, worker_thread, (void*)(uintptr_t)i);
    }

    /* ── Wait for all threads to end ── */
    for (int i = 0; i < 4; i++) {
        pthread_join(tid[i], NULL);
    }
    pthread_mutex_destroy(&s_lock);
    return 0;
}

What Are Registers?

Registers are the CPU's fastest storage — named slots inside the processor itself, holding values currently being worked on. They are orders of magnitude faster than RAM (0 cycle latency vs dozens). The compiler's primary job during code generation is deciding which variables live in registers and which get spilled to the stack. Understanding registers explains why C code behaves the way it does at runtime.

REGISTER VS RAM SPEED Register access: 0 cycles (part of ALU pipeline)
L1 cache: ~4 cycles
L2 cache: ~12 cycles
RAM (DRAM): ~100–200 cycles

On a Cortex-M4 at 168MHz: a RAM read costs ~600ns. A register operation costs ~6ns. The compiler works hard to keep hot variables in registers.
HOW THE COMPILER USES REGISTERS At -O0: every variable is spilled to the stack immediately — easy to debug but slow.

At -O2: the compiler runs register allocation (graph colouring algorithm) to keep as many variables in registers as possible, only spilling when it runs out of registers.

Special-Purpose Registers — Deep Dive
SP — Stack Pointer (r13)

Tracks the top of the current stack. On ARM, the stack grows downward — SP decreases on push, increases on pop. Every function prologue subtracts from SP to reserve space for locals; the epilogue adds it back. SP must always be 4-byte aligned (8-byte on function calls per AAPCS).

    int x = 10, y = 20;
    return x + y + a;
}

/* ARM Thumb assembly (-O0): */
// foo:                           ; on entry SP = 0x20008000 */
//   PUSH {r7, lr}                ; SP = SP - 8 = 0x20007FF8 (save r7 and LR)
//   SUB  sp, sp, #8              ; SP = SP - 8 = 0x20007FF0 (space for x and y)
//   ADD  r7, sp, #0              ; r7 = frame pointer = 0x20007FF0
//   STR  r0, [r7, #4]            ; store arg a at [SP+4]
//   MOV  r3, #10

//   MOV  r3, #20

//   ADD  sp, r7, #8              ; SP restored

LR — Link Register (r14)

When the CPU executes a BL (Branch with Link) instruction to call a function, it automatically saves the return address (PC + 4) into LR. The called function returns by writing LR back into PC. If the function calls another function, LR gets overwritten — so the compiler must push LR onto the stack first.

/* Leaf function (calls nobody) — LR never needs saving: */
int add(int a, int b) { return a + b; }
// add:
//   ADD r0, r0, r1    ; compute result in r0
//   BX  lr            ; jump to address in LR — no stack needed

/* Non-leaf function (calls add) — LR must be saved: */
int compute(int x) {
    return add(x, 5) * 2;
}
// compute:
//   PUSH {lr}         ; save LR — BL add will overwrite it
//   MOV  r1, #5
//   BL   add          ; LR = address of next instruction here; jumps to add
//   LSL  r0, r0, #1   ; r0 = result * 2
//   POP  {pc}         ; restore saved LR into PC → return to caller

/* ISR (Interrupt Service Routine) — special LR value EXC_RETURN: */
// On exception entry, CPU pushes {r0-r3,r12,LR,PC,xPSR} automatically
// LR is set to a special EXC_RETURN value (e.g. 0xFFFFFFF9)
// BX lr from ISR → CPU detects EXC_RETURN value → exception return sequence
PC — Program Counter (r15)

Always holds the address of the next instruction to fetch. On ARM, PC reads return current address + 4 (pipelining). Writing to PC causes an unconditional jump. The CPU increments PC automatically after each fetch; branches and calls do so explicitly.

xPSR — Program Status Register

A composite register combining three views: Application PSR (APSR), Interrupt PSR (IPSR), and Execution PSR (EPSR). The ALU flags live in APSR and are set automatically by arithmetic instructions ending in S (e.g. ADDS, SUBS).

/* xPSR bit layout (32-bit): */
/* [31] N — Negative flag: set if result was negative           */
/* [30] Z — Zero flag:     set if result was zero               */
/* [29] C — Carry flag:    set on unsigned overflow/borrow      */
/* [28] V — Overflow flag: set on signed overflow               */
/* [27] Q — Saturation:    set by saturating arithmetic (DSP)   */
/* [26:25] IT — If-Then execution state for Thumb-2             */
/* [8:0]   ISR number: 0=thread mode, nonzero=exception number  */

// ... if-body ...
// .Lelse:

Caller-Saved vs Callee-Saved — What the Compiler Does

The ARM AAPCS (Procedure Call Standard) divides registers into two classes. This determines what assembly the compiler emits in every function prologue and epilogue, and what it must do before any function call.

CALLER-SAVED: r0–r3, r12 (scratch) The calling function must save these if it still needs their values after a call — the called function is free to destroy them.

What the compiler does: if r0–r3 hold live values before a BL, it spills them to the stack before the call and reloads after.

Benefit: leaf functions and simple callees never need to save/restore — faster function calls.
CALLEE-SAVED: r4–r11, SP (preserved) The called function must restore these to their original values before returning — the caller can rely on them surviving any call.

What the compiler does: if a function uses r4–r11, it pushes them in the prologue and pops in the epilogue.

Benefit: variables in r4–r11 survive across calls without the caller doing anything. 
/* Concrete example — what the compiler generates: */
int outer(int x, int y) {
    int saved = x * 2;         /* compiler puts 'saved' in r4 (callee-saved) */
    int result = inner(y, 3); /* inner() can trash r0-r3, not r4            */
    return saved + result;     /* r4 still has correct 'saved' value        */
}

// outer:                      ; prologue
//   PUSH {r4, lr}             ; save r4 (will use it) and LR (will call inner)
//   LSL  r4, r0, #1           ; r4 = x*2 = 'saved' — safe across inner() call
//   MOV  r1, #3               ; arg2 for inner = 3
//                             ; r0 already = y (arg1 for inner)
//   BL   inner                ; call — r0-r3 may be trashed, r4 survives
//   ADD  r0, r4, r0           ; return saved + inner_result
//   POP  {r4, pc}             ; restore r4, return

What Are Serial Protocols?

Serial communication sends data one bit at a time over a wire — as opposed to parallel buses that send multiple bits simultaneously. In embedded systems, three protocols dominate peripheral communication: UART, I2C, and SPI. Each makes a different trade-off between pin count, speed, number of devices, and complexity.

UART

2-wire, async, point-to-point. No shared clock. Each side agrees on baud rate in advance. Used for debug consoles, GPS modules, Bluetooth modules.

I2C

2-wire, sync, multi-device bus. One master drives a shared clock. Up to 127 devices on same 2 wires. Used for sensors, EEPROMs, display drivers.

SPI

4-wire, sync, full-duplex. Fastest of the three. One master, multiple slaves via chip-select lines. Used for flash memory, ADCs, displays, SD cards.

How UART Transmission Works

UART is asynchronous — there is no clock wire. The line idles at logic HIGH (1). A transmission begins when the sender pulls the line LOW for exactly one bit-period — this is the Start Bit. The receiver, seeing a falling edge, starts its own timer and samples the line at the center of each subsequent bit-period. The baud rate determines how long each bit lasts.

Parity Bit — Even and Odd, with Examples

A parity bit is appended after the data bits as a simple single-bit error detection mechanism. It can detect any single-bit error in the frame (but not 2-bit errors). The parity bit is chosen to make the total number of 1s in the data+parity either even or odd, depending on the configured mode.

EVEN PARITY The parity bit is chosen so the total count of 1s (data + parity) is even. 
/* Data: 0x41 = 0b01000001 */
/* Count of 1s in data: 2  */
/* 2 is already even       */
/* → Even parity bit = 0   */
/* Frame: 01000001 P=0     */
/* Total 1s: 2 (even ✓)   */

/* Data: 0x42 = 0b01000010 */
/* Count of 1s: 2 (even)   */
/* → Even parity bit = 0   */

/* Data: 0x43 = 0b01000011 */
/* Count of 1s: 3 (odd)    */
/* → Even parity bit = 1   */
/* Frame: 01000011 P=1     */
/* Total 1s: 4 (even ✓)   */
ODD PARITY The parity bit is chosen so the total count of 1s (data + parity) is odd. 
/* Data: 0x41 = 0b01000001 */
/* Count of 1s: 2 (even)   */
/* Need odd total           */
/* → Odd parity bit = 1    */
/* Frame: 01000001 P=1     */
/* Total 1s: 3 (odd ✓)    */

/* Data: 0x43 = 0b01000011 */
/* Count of 1s: 3 (odd)    */
/* Already odd total        */
/* → Odd parity bit = 0    */
/* Frame: 01000011 P=0     */
/* Total 1s: 3 (odd ✓)    */
/* Computing parity in C: */
uint8_t even_parity(uint8_t data) {
    data ^= data >> 4;
    data ^= data >> 2;
    data ^= data >> 1;
    return data & 1;   /* 0 = even number of 1s, 1 = odd number of 1s */
}
/* For even parity bit: use return value directly               */
/* For odd  parity bit: use return value XOR 1 (flip the bit)  */
uint8_t ep = even_parity(0x43);   /* ep=1 → even parity bit is 1 */
uint8_t op = ep ^ 1;               /* op=0 → odd  parity bit is 0 */

/* Parity limitation: two simultaneous bit flips cancel out — undetected! */
/* 01000001 → 00000011 (bits 6 and 0 flipped) → same parity, wrong data  */
CRC — Cyclic Redundancy Check

CRC is a much stronger error detection mechanism used in UART protocols (like Modbus) and packet-based systems. The transmitter computes a CRC value over the entire payload and appends it. The receiver recomputes the CRC over the received data and compares — any mismatch means corruption. CRC detects all single-bit, double-bit, and most burst errors.

HOW CRC WORKS CRC treats the message as a large binary number and divides it by a fixed generator polynomial. The remainder of that division is the CRC value. Both sides use the same polynomial. Common polynomials: CRC-8 (1 byte), CRC-16 (2 bytes, Modbus), CRC-32 (4 bytes, Ethernet/ZIP). 
/* ── CRC-8 (polynomial 0x07) example — manual computation ── */
uint8_t crc8(uint8_t *data, size_t len) {
    uint8_t crc = 0x00;
    for (size_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (int bit = 0; bit < 8; bit++) {
            if (crc & 0x80)
                crc = (crc << 1) ^ 0x07;  /* XOR with polynomial */
            else
                crc = (crc << 1);
        }
    }
    return crc;
}

/* ── Concrete example ── */
/* Payload: {0x01, 0x02, 0x03}                          */
/* CRC-8 result: 0x48                                   */
/* Transmitted frame: 0x01 0x02 0x03 0x48               */
/*                                    ^^^^— CRC appended */

/* Receiver: */
uint8_t frame[]   = {0x01, 0x02, 0x03, 0x48};
uint8_t rx_crc    = frame[3];                   /* last byte = CRC */
uint8_t calc_crc  = crc8(frame, 3);             /* compute over payload only */
if (rx_crc != calc_crc) { /* error! */ }

/* ── CRC-16 (Modbus, polynomial 0x8005): */
uint16_t crc16_modbus(uint8_t *data, size_t len) {
    uint16_t crc = 0xFFFF;
    for (size_t i = 0; i < len; i++) {
        crc ^= (uint16_t)data[i];
        for (int bit = 0; bit < 8; bit++) {
            if (crc & 0x0001)
                crc = (crc >> 1) ^ 0xA001;
            else
                crc = (crc >> 1);
        }
    }
    return crc;  /* append LSB first in Modbus frame */
}
Read/Write Cycle & Clock Signal Timing

Because UART is asynchronous, the "clock" is virtual — each side independently runs a baud rate timer. The receiver detects the falling edge of the start bit, then waits 1.5 bit-periods to sample in the center of bit D0, then every 1 bit-period thereafter.

/* SDA: ‾‾|addr bits   |R/W|ACK|data bits|ACK| */
/*       ↑START                              ↑STOP */
/*                                             */
/* SDA must be STABLE while SCL is HIGH (data valid)  */
/* SDA may only CHANGE while SCL is LOW (transition)  */
/* Exception: START = SDA falls while SCL is HIGH     */
/* Exception: STOP  = SDA rises while SCL is HIGH     */

/* ── ACK/NACK ── */
/* After every 8 data bits, master releases SDA (pulls high via pull-up) */
/* If slave received OK → slave pulls SDA LOW = ACK (0)                  */
/* If slave busy/error  → slave leaves SDA HIGH = NACK (1)               */
/* Master reads SDA on the 9th SCL pulse to check ACK/NACK               */
/* ── STM32 HAL I2C Write ── */
uint8_t data[2] = {0x00, 0xAB};   /* register address + value */
HAL_StatusTypeDef ret = HAL_I2C_Master_Transmit(
    &hi2c1,          /* I2C handle              */
    0x48 << 1,       /* slave addr, shifted left */
    data, 2,          /* buffer and length        */
    100               /* timeout ms              */
);
if (ret != HAL_OK) { /* NACK received or bus error */ }

/* ── STM32 HAL I2C Read (e.g. read temperature register) ── */
uint8_t reg = 0x00;
uint8_t rx[2];
HAL_I2C_Master_Transmit(&hi2c1, 0x48<<1, ®, 1, 100); /* send reg addr */
HAL_I2C_Master_Receive (&hi2c1, 0x48<<1, rx,  2, 100); /* read 2 bytes  */
int16_t temp = (rx[0] << 8) | rx[1];              /* combine bytes */
I2C Read/Write Cycle — Repeated START

To read from a register, I2C requires a write phase (tell the device which register) followed immediately by a read phase. These are joined with a Repeated START (Sr) — a new START without a STOP, keeping bus ownership and preventing another master from taking over.

/* Full read sequence: Write register address, then Read data */
/*                                                             */
/* [START][0x48 W][ACK][reg=0x00][ACK][Sr][0x48 R][ACK][data MSB][ACK][data LSB][NACK][STOP] */
/*                                           ↑                                    ↑          */
/*                               Repeated START (no STOP)      Master sends NACK on last byte */
/*                                                                                             */
/* Master sends NACK on the last byte it wants to receive — tells slave to stop.  */
/* Then master issues STOP condition.                                             */

/* HAL does this automatically with Master_Mem_Read: */
HAL_I2C_Mem_Read(&hi2c1, 0x48<<1, 0x00, I2C_MEMADD_SIZE_8BIT, rx, 2, 100);

SPI uses 4 wires: SCLK (clock), MOSI (Master Out Slave In), MISO (Master In Slave Out), and CS/NSS (Chip Select, active LOW). Unlike I2C, SPI is full-duplex — data flows in both directions simultaneously. Each slave gets its own CS line, so adding slaves costs one pin per slave.

SPI Frame & Clock Modes — CPOL / CPHA

SPI has no fixed framing — it is just a shift register. The master shifts out bits on MOSI while simultaneously shifting in bits on MISO, driven by SCLK. The key configuration is CPOL (clock polarity: idle level) and CPHA (clock phase: when data is sampled). These combine into 4 modes.

SPI Read/Write Cycle — Clock, CS, and Timing

The master controls CS and SCLK entirely. There are no ACKs — if the slave isn't ready it can't signal back (unlike I2C). Some devices use a BUSY line (extra GPIO) for this. Data width is configurable: 8, 16, or 32 bits per transfer, and the master can keep CS low to chain multiple bytes in one burst.

/* ── Multi-byte burst read from W25Q flash (SPI Mode 0) ── */
/* Command: 0x03 (READ), then 3 address bytes, then N data bytes */
uint8_t cmd[4] = {0x03, 0x00, 0x10, 0x00};  /* READ at addr 0x001000 */
uint8_t data[64];

HAL_GPIO_WritePin(CS_GPIO_Port, CS_Pin, GPIO_PIN_RESET);
HAL_SPI_Transmit(&hspi2, cmd, 4, 10);       /* send command+address */
HAL_SPI_Receive(&hspi2, data, 64, 100);     /* read 64 bytes        */
HAL_GPIO_WritePin(CS_GPIO_Port, CS_Pin, GPIO_PIN_SET);

/* Raising CS mid-transfer aborts the operation on most flash devices       */

⚠ COMMON PITFALLS UART: Baud rate mismatch causes garbled data. Always verify APB clock and BRR register. Use oscilloscope if in doubt.

Algorithm Efficiency & Big-O Notation
// TIME COMPLEXITY · SPACE COMPLEXITY · BEST · AVERAGE · WORST CASE
What Is Big-O Notation?

Big-O notation describes how the runtime or memory usage of an algorithm scales as the input size n grows toward infinity. It expresses the upper bound — the worst-case growth rate — stripping away constants and lower-order terms. It answers: "if I double the input, what happens to the cost?"

THREE CASES Best case (Ω): most favourable input. Rarely useful in practice.

Average case (Θ): expected cost over all possible inputs. Most useful for real-world analysis.

Worst case (O): guaranteed upper bound regardless of input. Used for safety-critical and embedded systems.
COMPLEXITY HIERARCHY (slowest to fastest growth) O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)

For n = 1000: O(1)=1, O(log n)≈10, O(n)=1000, O(n log n)≈10000, O(n²)=1,000,000, O(2ⁿ)=astronomical.
O(1) — Constant Time Best possible

The operation takes the same amount of time regardless of input size. No loops, no recursion proportional to n. Adding more data has zero effect on execution time.

/* ── Array index access — always 1 step ── */
int get_element(int *arr, int idx) {
    return arr[idx];    /* pointer + offset — one memory read, always O(1) */
}
/* arr[0] or arr[1000000] — identical cost */

/* ── Stack push/pop — always 1 step ── */
void push(Stack *s, int val) {
    s->data[s->top++] = val;   /* one write + one increment */
}

/* ── Hash table lookup (ideal) ── */
int val = hashtable_get(ht, key);  /* hash(key) → index → read: O(1) avg */

/* ── Bitmask operations ── */
uint32_t flags = 0xDEAD;
int is_set = (flags >> 5) & 1;   /* always 2 instructions — O(1) */

/* ── Swap two variables ── */
void swap(int *a, int *b) {
    int tmp = *a; *a = *b; *b = tmp;  /* 3 ops regardless of value size */
}
✓ REAL EXAMPLES Array access, stack push/pop, linked list head insert/delete, hash table get/set (average), any math formula on fixed-width integers, reading a hardware register.
O(log n) — Logarithmic Time Excellent

The algorithm halves the problem (or divides by some constant factor) at each step. Doubling the input adds only one more step. Base of the logarithm doesn't matter for Big-O purposes — log₂n and log₁₀n differ by only a constant.

/* ── Binary search — halves search space each iteration ── */
int binary_search(int *arr, int n, int target) {
    int lo = 0, hi = n - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;   /* avoids overflow vs (lo+hi)/2 */
        if      (arr[mid] == target) return mid;
        else if (arr[mid] <  target) lo = mid + 1;
        else                         hi = mid - 1;
    }
    return -1;   /* not found */
}
/* n=1000:   max 10 iterations  (log₂ 1000 ≈ 10) */
/* n=1000000: max 20 iterations (log₂ 1M   ≈ 20) */
/* Requires: array must be SORTED beforehand      */

/* ── Balanced BST search — same halving logic ── */
int bst_search(Node *root, int target) {
    if (!root)              return -1;
    if (root->val == target) return  1;
    if (target < root->val)  return bst_search(root->left,  target);
    else                    return bst_search(root->right, target);
}
/* Each call eliminates half the tree — O(log n) for balanced tree */
/* Degrades to O(n) if tree is unbalanced (e.g. sorted insertions)  */

/* ── Integer exponentiation by squaring ── */
long power(long base, int exp) {
    long result = 1;
    while (exp > 0) {
        if (exp & 1) result *= base;
        base *= base;
        exp >>= 1;         /* halve exponent each step */
    }
    return result;
}
/* power(2, 32): 5 squarings instead of 32 multiplications */
WHY log n IS NEARLY AS GOOD AS O(1) For n = 4 billion (2³²), log₂n = 32 steps. Binary search on a 4-billion-element sorted array finds any element in at most 32 comparisons. This is why sorted arrays + binary search are preferred over linear scans in embedded lookup tables.
O(n) — Linear Time Good — unavoidable for unsorted data

The algorithm visits each element once (or a constant number of times). Doubling input doubles runtime. This is the minimum possible for algorithms that must examine every element.

/* ── Linear search — must check each element ── */
int linear_search(int *arr, int n, int target) {
    for (int i = 0; i < n; i++) {
        if (arr[i] == target) return i;
    }
    return -1;
}
/* Best case: O(1) — found at index 0   */
/* Worst case: O(n) — not found at all  */
/* Average case: O(n/2) = O(n)          */

/* ── Sum of array ── */
long array_sum(int *arr, int n) {
    long sum = 0;
    for (int i = 0; i < n; i++) sum += arr[i];
    return sum;   /* n additions — O(n) */
}

/* ── Find maximum ── */
int find_max(int *arr, int n) {
    int max = arr[0];
    for (int i = 1; i < n; i++)
        if (arr[i] > max) max = arr[i];
    return max;   /* must check every element — O(n) */
}

/* ── Reverse an array in-place ── */
void reverse(int *arr, int n) {
    for (int l = 0, r = n-1; l < r; l++, r--) {
        int tmp = arr[l]; arr[l] = arr[r]; arr[r] = tmp;
    }
}
/* n/2 swaps = O(n) — constants dropped */

/* ── Count occurrences (frequency map) ── */
void count_freq(int *arr, int n, int *freq, int range) {
    memset(freq, 0, range * sizeof(int));
    for (int i = 0; i < n; i++) freq[arr[i]]++;  /* O(n) */
}
O(n log n) — Linearithmic Time Good — optimal for comparison sorts

The theoretical minimum for any comparison-based sorting algorithm. The n log n factor comes from dividing the problem log n times (like binary) while still needing to process all n elements at each level. Mergesort and Heapsort always achieve this. Quicksort achieves it on average.

/* ── Merge sort — always O(n log n) ── */
void merge(int *arr, int l, int m, int r) {
    int n1 = m - l + 1, n2 = r - m;
    int L[n1], R[n2];
    for (int i = 0; i < n1; i++) L[i] = arr[l + i];
    for (int j = 0; j < n2; j++) R[j] = arr[m + 1 + j];
    int i = 0, j = 0, k = l;
    while (i < n1 && j < n2)
        arr[k++] = (L[i] <= R[j]) ? L[i++] : R[j++];
    while (i < n1) arr[k++] = L[i++];
    while (j < n2) arr[k++] = R[j++];
}
void merge_sort(int *arr, int l, int r) {
    if (l < r) {
        int m = l + (r - l) / 2;
        merge_sort(arr, l, m);        /* T(n/2) */
        merge_sort(arr, m + 1, r);   /* T(n/2) */
        merge(arr, l, m, r);          /* O(n)   */
    }
}
/* Recurrence: T(n) = 2T(n/2) + O(n) → solves to O(n log n) */
/* log n levels of recursion × n work per level = n log n    */
/* Guaranteed O(n log n) worst case — preferred for embedded  */

/* ── Quicksort — O(n log n) average, O(n²) worst ── */
int partition(int *arr, int lo, int hi) {
    int pivot = arr[hi], i = lo - 1;
    for (int j = lo; j < hi; j++)
        if (arr[j] <= pivot) { i++; int t=arr[i]; arr[i]=arr[j]; arr[j]=t; }
    int t = arr[i+1]; arr[i+1] = arr[hi]; arr[hi] = t;
    return i + 1;
}
void quicksort(int *arr, int lo, int hi) {
    if (lo < hi) {
        int p = partition(arr, lo, hi);
        quicksort(arr, lo, p - 1);
        quicksort(arr, p + 1, hi);
    }
}
/* Average O(n log n), but worst case is O(n²) with sorted input + bad pivot */
/* Fix: randomize pivot, or use median-of-3 pivot selection                  */

/* C stdlib: qsort() — typically introsort (quicksort + heapsort fallback) */
qsort(arr, n, sizeof(int), compare_fn);  /* O(n log n) guaranteed     */
WHY n log n IS THE SORTING LOWER BOUND Any comparison-based sort must distinguish between n! possible orderings. Each comparison gives 1 bit of information. Bits needed = log₂(n!) ≈ n log n (Stirling's approximation). You cannot sort by comparisons faster than O(n log n).
O(n²) — Quadratic Time Acceptable for small n only

Two nested loops each iterating over n elements. Doubling input quadruples runtime. Fine for n < 1000; painful for n = 10000; unusable for n = 1,000,000.

/* ── Bubble sort — nested loops, O(n²) ── */
void bubble_sort(int *arr, int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j+1]) {
                int t = arr[j]; arr[j] = arr[j+1]; arr[j+1] = t;
            }
        }
    }
}
/* n=100:    ~5,000 comparisons */
/* n=1000:   ~500,000 comparisons */
/* n=10000:  ~50,000,000 comparisons — noticeably slow */

/* ── Insertion sort — O(n²) worst, O(n) best (nearly sorted input) ── */
void insertion_sort(int *arr, int n) {
    for (int i = 1; i < n; i++) {
        int key = arr[i], j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j+1] = arr[j];
            j--;
        }
        arr[j+1] = key;
    }
}
/* Best for nearly-sorted small arrays — used inside timsort for small runs */

/* ── Selection sort — always O(n²), no best case improvement ── */
void selection_sort(int *arr, int n) {
    for (int i = 0; i < n-1; i++) {
        int min_idx = i;
        for (int j = i+1; j < n; j++)
            if (arr[j] < arr[min_idx]) min_idx = j;
        int t = arr[i]; arr[i] = arr[min_idx]; arr[min_idx] = t;
    }
}

/* ── All pairs — check every combination ── */
void print_pairs(int *arr, int n) {
    for (int i = 0; i < n; i++)
        for (int j = i+1; j < n; j++)
            printf("(%d,%d)\n", arr[i], arr[j]);   /* n*(n-1)/2 pairs = O(n²) */
}

/* ── Matrix multiplication — naive: O(n³), Strassen: O(n^2.807) ── */
void matmul(int A[N][N], int B[N][N], int C[N][N]) {
    for (int i = 0; i < N; i++)
        for (int j = 0; j < N; j++) {
            C[i][j] = 0;
            for (int k = 0; k < N; k++)
                C[i][j] += A[i][k] * B[k][j];  /* O(n³) */
        }
}
O(2ⁿ) — Exponential & O(n!) — Factorial Avoid for large n

These complexities explode so fast they are only feasible for very small n (typically n < 20 for 2ⁿ, n < 12 for n!). They arise naturally from problems that explore all subsets or all permutations.

/* ── O(2ⁿ): Naive recursive Fibonacci ── */
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);  /* 2 calls each level — 2ⁿ total calls */
}
/* fib(10)  → 177 calls                  */
/* fib(30)  → 2.7 million calls          */
/* fib(50)  → 2²⁵ × overhead ≈ unusable  */
/* Fix: memoization or iteration → O(n)  */

/* ── O(2ⁿ): All subsets of a set ── */
void all_subsets(int *set, int n) {
    for (int mask = 0; mask < (1 << n); mask++) {  /* 2ⁿ masks */
        for (int i = 0; i < n; i++)
            if (mask & (1 << i)) printf("%d ", set[i]);
        printf("\n");
    }
}
/* n=3: 8 subsets. n=20: 1,048,576 subsets. n=30: 1 billion+ */

/* ── O(n!): Generate all permutations ── */
void permutations(int *arr, int start, int n) {
    if (start == n) { /* print arr */ return; }
    for (int i = start; i < n; i++) {
        int t = arr[start]; arr[start] = arr[i]; arr[i] = t;
        permutations(arr, start+1, n);
        t = arr[start]; arr[start] = arr[i]; arr[i] = t;
    }
}
/* n=3: 6 perms. n=10: 3,628,800. n=13: 6.2 billion — too slow */
/* NP-hard problems like TSP (Travelling Salesman) are O(n!)    */
⚠ MEMOIZATION TRANSFORMS O(2ⁿ) → O(n) Cache results you've already computed. For Fibonacci: store computed values in an array and return immediately on repeat calls. This turns 2.7 million calls for fib(30) into exactly 30 calls — a 90,000× speedup.
Space Complexity — Memory Cost

Space complexity measures how much extra memory an algorithm needs relative to input size. Always analyse time and space — some algorithms trade one for the other. Space complexity counts only auxiliary memory, not the input itself.

/* ── O(1) space: in-place operations ── */
void reverse_inplace(int *a, int n) {
    for (int l=0, r=n-1; l<r; l++,r--) {
        int t=a[l]; a[l]=a[r]; a[r]=t;   /* only 1 extra var regardless of n */
    }
}

/* ── O(n) space: copy / auxiliary array ── */
int* array_copy(int *src, int n) {
    int *dst = (int*)malloc(n * sizeof(int));   /* n extra slots */
    memcpy(dst, src, n * sizeof(int));
    return dst;
}
/* Merge sort also uses O(n) auxiliary space for temp arrays */

/* ── O(log n) space: recursion stack depth ── */
/* Binary search iterative → O(1) space                */
/* Binary search recursive → O(log n) call stack frames */
int bsearch_rec(int *a, int lo, int hi, int t) {
    if (lo > hi) return -1;
    int mid = lo + (hi-lo)/2;
    if (a[mid]==t) return mid;
    if (a[mid]<t)  return bsearch_rec(a, mid+1, hi, t);  /* log n frames */
    else           return bsearch_rec(a, lo, mid-1, t);
}

✓ EMBEDDED GUIDELINE On a Cortex-M4 at 168MHz, you get roughly 168 million simple operations per second. For a time-critical ISR that must finish in 10µs, you have ~1680 operations. This means O(n²) with n=40 is borderline, O(n) with n=1000 is fine, O(log n) with any reasonable n is always safe.
⚠ CONSTANTS MATTER IN PRACTICE Big-O ignores constants. O(n) with a large constant can be slower than O(n²) for small n. Insertion sort (O(n²)) is faster than mergesort (O(n log n)) for n < ~16 due to cache effects and smaller constants. Always profile before optimizing.
